# Changelog

All notable changes to iHotel are documented here.

---

## [Unreleased] — 2026-03-24

### Added
- **Reserved room status** — rooms with an active reservation now display as **Reserved** (cyan) in the heatmap, room table, room modal, and hover tooltips. Separate from the physical NOT_OCCUPIED device state; computed in the UI only (`roomStatus === 0 && reservation != null`)
- **Checkout date in room modal** — when a room is not Vacant or Maintenance and has an active reservation, the checkout date is shown as a badge in the room modal header (visible to all roles including guests)
- `status_reserved` i18n key (English: "Reserved" / Arabic: "محجوزة"); `status_not_occupied` corrected to "Not Occupied"
- **Reserved filter** in Room Table — filter bar now includes a "Reserved" option showing only rooms with pending reservations
- `VITE_GUEST_HOST` env variable in `client/.env` — overrides the hostname used in QR code URLs when the dashboard is opened via localhost but guests should reach the LAN IP

### Changed
- **Reservation no longer sets NOT_OCCUPIED** — creating a reservation (staff PMS or self-booking) no longer pushes `roomStatus: 4` to the device. Room stays VACANT until the guest physically arrives
- **Room reset now cancels active reservation** — resetting a room via the Reset button cancels any active reservation for that room and broadcasts a lockout event, so the room correctly returns to Vacant
- **Auto-OCCUPIED on door open for reserved rooms** — improved reliability: uses `lastOverview.reservation` instead of device-reported status, so it fires even if device telemetry hasn't confirmed `roomStatus: 0` yet
- **Auto-OCCUPIED on lights/AC/curtains for reserved rooms** — physical activity (line1/line2/line3 on, AC mode > 0, curtains/blinds moved) in a reserved room now also triggers automatic transition to OCCUPIED
- `CORS_ORIGIN` and `GUEST_URL_BASE` in `server/.env` updated from ngrok URL to local network IP (`http://192.168.1.10:5173`)
- `status_not_occupied` label corrected from "Reserved" to "Not Occupied" to avoid confusion with the new Reserved display state

### Fixed
- QR codes generated by self-booking (`/book/:slug`) were encoding `localhost` as the host because `GUEST_URL_BASE` was set to an expired ngrok URL — now uses LAN IP
- `better-sqlite3` native module rebuilt against Node.js v22 after version mismatch (`NODE_MODULE_VERSION 120 → 127`)
- Port 3000 `EADDRINUSE` on startup — stale server process from previous run; resolved by killing the leftover process

## [2.0.0] — Prior release

- Multi-tenant SaaS platform (hotel provisioning via platform super-admin)
- Self-booking public pages (`/book/:slug`) with image galleries, availability, and payment flow
- Hotel profile management (owner tab): hero image, room type photos, description, amenities, online booking toggle
- Kiosk display page (`/kiosk/:slug`) for lobby screens
- Payment flow and booking UX improvements
- Vite dev server proxying `/uploads` through to backend
